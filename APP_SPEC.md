# Scratchpad
Purpose: for computer workers who need to jot a quick note or draft right now — a completely blank, instantly-typeable markdown page that formats itself as you type and keeps every note in a quiet, chrome-free left column of text — with nothing to set up.
Problem: software engineers, PMs, analysts, and marketers reach for a throwaway place to paste a snippet, draft a Slack message, or outline a thought several times a day. The usual fallbacks — a new untitled doc, a Notion page, a sticky-note app — make you pick a location, name a file, or stare at a toolbar before you can type. They want the cursor to already be blinking.
Beats alternative: vs Notion/Google Docs/Apple Notes, this app drops you ON a blinking cursor with ZERO setup (no new-doc dialog, no naming a file, no login) and live-formats markdown shortcuts (Dropbox-Paper style "## "→heading) as you type, so the very first keystroke is real writing — and a quiet, lines-free column of faint-grey note titles keeps every throwaway note you make, most-recent first, one click apart. That instant blank-canvas-with-live-markdown plus a frictionless, calm note list is the reason this app exists.
Core flows:
1. New note → type immediately (instant-typing delight, every note). A quiet left sidebar (bare faint-grey text, no panel chrome) has a "+ new note" action at the top; clicking it creates a fresh blank note, makes it active, and focuses the caret so the first keystroke is real writing (zero clicks to type). First-ever load with no saved notes lands directly in a fresh blank focused note (original cold-open delight). When the user starts typing on a NEW blank note, today's date is auto-inserted ONCE as a quiet dateline directly ABOVE the first typed line, tight with NO blank line/gap underneath (format "Saturday, June 20" — weekday, full month, day, no year/ordinal; captured at first keystroke, does not live-update, editable/deletable content, new notes only). The dateline renders very faint (--grey-600) and small — body-size or smaller, never a heading.
2. Live markdown input-rules as you type (the differentiator). Markdown shortcuts transform the current line/selection in place, consuming the syntax: "# " / "## " / "### " → H1/H2/H3; "- " → bullet; "1. " → numbered list; "> " → blockquote; "`code`" → inline code; "**bold**" → bold; "*italic*" → italic; "---" → horizontal divider. Pressing Enter after a heading returns the next line to a normal paragraph. A live character count sits in the lower-right of the writing surface, updating as you type.
3. Multi-note bare-text sidebar + autosave (per-note localStorage). Each note is its own localStorage record (id, content, updatedAt); autosave is debounced and OFF the keystroke path. The sidebar is a chrome-free column of faint-grey text — NO dividers, borders, background fill, or box — floating on the same paper as the editor. It lists every saved note most-recently-edited first; "+ new note" sits at the top in lowercase, and every row below shares ONE consistent text treatment (same typeface/size/weight/faint-grey). Each row's title derives from the first few phrases of the note's actual CONTENT that fit within ~20 chars (truncated cleanly), SKIPPING the auto-dateline line; fallback "Untitled" when there's no content yet. Click a row to open/switch (the current note is saved first); the ACTIVE row is marked by INK COLOR ONLY (active = full --ink, others faint grey) — no inversion, fill, shading, box, or pill. Delete a note via a quiet faint-grey "×" hover affordance that removes it IMMEDIATELY with NO confirm dialog. The last-open note is remembered across reloads. A whisper-quiet "Saved" affordance is the only save status shown.
4. Paste a photo into a note (Dropbox-Paper full-bleed). Pasting an image from the clipboard (Cmd/Ctrl+V) inserts it as a block-level, FULL-WIDTH image spanning the editor column (aspect-ratio preserved, height auto, comfortable vertical spacing above/below); the photo carries its own color while surrounding chrome stays monochrome. Image bytes are stored 100% client-side as a BLOB in IndexedDB keyed by id — the note's localStorage content references the image by id only (never base64-inlined into the localStorage record); on render the id resolves to an object URL (revoked on unmount). Very large pastes are auto-downscaled (re-encoded to a sane max dimension, ~2000px) before storing; deleting a note frees its images from IndexedDB. Pasted photos survive reload, and pasting does not break instant-typing delight or autosave. (Drag-and-drop of an image file is a nice-to-have; paste is the requirement.)
Success checks:
1. New-note focus, zero clicks: on first-ever load (no saved notes) OR immediately after clicking "+ new note", WITHOUT clicking anywhere else, press the "h" key — "h" appears in the note and a text caret is visible. (Each new note is auto-focused.)
2. Heading rule: type exactly `## Roadmap` then no further typing — the line renders as a larger heading reading "Roadmap" with the leading "## " no longer visible anywhere on the page.
3. Bullet rule: on a new empty line type exactly `- buy milk` — the line renders as a bulleted list item showing a bullet marker followed by "buy milk", with "- " no longer visible as literal text.
4. Bold rule: type exactly `**ship it**` — the words "ship it" render in bold weight and the literal asterisks are no longer visible.
5. Divider rule: on a new empty line type exactly `---` — a horizontal rule (divider line) renders in place of the three dashes.
6. No layout jump: with the caret on a paragraph line, type `## ` to convert it to a heading — surrounding lines do not jump position (the line reformats in place; only that line changes size).
7. Comfortable measure: the writing column is centered with a constrained max width (roughly 65–75 characters per line, not full-bleed edge-to-edge).
8. Grotesque typeface: the editor writing surface and all chrome (sidebar, saved indicator, char count, dateline) render in the SSENSE grotesque stack ("Helvetica Neue" → Archivo fallback), not a serif font.
9. SIDEBAR — new note + instant typing: clicking "+ new note" adds a new row to the sidebar list, switches to it (it becomes the active row), and lands the focused caret in a blank surface so the next keystroke is writing — and the click does NOT get eaten by a prior surface's autofocus (the first keystroke after the click lands in the new note, not the old one).
10. SIDEBAR — title from CONTENT (not dateline), ≤~20 chars + most-recent ordering: type `meeting notes` as the first typed line of a note (which auto-stamps a dateline above) — its sidebar row title reads "meeting notes" (derived from the content line, NOT the date), truncated to ~20 chars if longer; a note with no typed content shows "Untitled". Editing the first content line updates the title. The most-recently-edited note appears at the top of the list.
11. SIDEBAR — bare text, no chrome, active-by-ink: the sidebar has NO visible divider/border/background/box (it is faint-grey text on the same paper); "+ new note" is lowercase; the currently open note's row is marked ONLY by ink color (active row = full --ink, all other rows + "+ new note" = faint grey) with no inversion, fill, shading, pill, or rounded box; exactly one row is active at a time.
12. SIDEBAR — switch saves & restores (returning-user regression): create note A with `alpha note`, create note B with `bravo note`, click back to A's row — A opens showing "alpha note" (B's edits were saved first, A was not lost); reload the page — the last-open note re-opens and BOTH rows are still present in the list. (Covers multi-record persistence for a returning user, not just clean cold-open.)
13. SIDEBAR — delete with NO confirm: hovering a note row reveals a quiet faint-grey "×" affordance; activating it removes that row from the list and from localStorage IMMEDIATELY with NO "Delete this note?" dialog/confirm step; the other notes remain.
14. CHAR COUNT — live + styled: with text in the note, a character count is visible in the LOWER-RIGHT of the writing surface and updates as you type (e.g. typing one more character increments it); it renders tiny, quiet, in --grey-600 with tabular-nums, monochrome (no fill/shadow).
15. DATELINE — auto on first keystroke, tight + faint, new notes only: click "+ new note", then type `meeting notes` — a quiet --grey-600 dateline reading today's date in the exact form "Saturday, June 20" (weekday, full month, day; NO year, no ordinal suffix) appears as the line directly ABOVE "meeting notes" with NO empty paragraph/blank gap between them, rendered at body size or smaller (not a heading); the dateline is editable/deletable real content. An untouched blank new note shows NO dateline until the first keystroke, and the dateline does not retroactively appear on previously-saved notes.
16. SNAPPY typing (off the keystroke path): typing does not serialize or persist the whole document on every keystroke — full-doc getJSON() / per-note save work happens only inside the debounced autosave timer; the per-keystroke path stays cheap (e.g. only the isEmpty boolean + character count update), so typing has no perceptible lag.
17. GLOBAL TYPE — thinner + ~20% smaller: the body/paragraph text renders at a lighter weight (one notch lighter than 400, ~300–350) and the whole UI is scaled down ~20% from its prior size (root font-size reduced ~16px→~13px so rem sizing scales uniformly); proportions/rhythm stay intact, text stays legible.
18. NO COPY affordance: there is NO copy icon, copy button, "copied"/"Copy blocked" confirmation, or any copy control anywhere in the UI on any note state (blank or non-empty), and the rich-text/clipboard copy code is removed from the built bundle (no ClipboardItem copy path remains).
19. MOBILE safe at 375px: at a 375px-wide viewport the sidebar collapses to a drawer (toggled by a quiet control) and the writing surface is full-width; when the drawer is open it does not permanently block the writing surface or controls, and when closed neither the drawer toggle nor the char count overlays the caret or causes horizontal overflow.
20. PASTE PHOTO — inserts a full-width image: with the caret in a note, pasting an image from the clipboard (a ClipboardEvent carrying an image/* item) inserts a block-level `<img>` into the writing surface that spans the FULL WIDTH of the editor column (its rendered width ≈ the measure/column width, not a small inline thumbnail), with aspect ratio preserved (height auto). Typing text before and after the image still works and the image sits on its own block line.
21. PASTE PHOTO — stored in IndexedDB, NOT base64 in localStorage: after pasting an image, the note's localStorage record does NOT contain a base64 `data:` image payload (it references the image by a short id); the image bytes live in IndexedDB. The image survives a full page reload (it re-resolves from IndexedDB to a visible `<img>` with a blob/object URL). Deleting the note removes its image blob from IndexedDB.
22. PASTE PHOTO — snappy + autosave intact + downscale: pasting a large image does not freeze the editor or block typing; a very large paste is downscaled before storage (stored dimension capped, not the raw multi-thousand-px original); autosave still persists the note (text + image reference) and the existing markdown/dateline/char-count flows are unaffected by the presence of an image.
Out of scope: no save/open dialogs or explicit note renaming (title auto-derives from content), no folders/tags/search across notes, no PDF/file export, NO copy affordance of any kind (rich-text or markdown copy fully removed per Elaine 2026-06-20), no image editing/captions/resize-handles (pasted photos render full-width as-is), no account or login, no server/Turso/cloud sync, no collaboration or sharing links, no toolbar or formatting buttons (formatting is keyboard markdown input-rules only). Notes are device-local in localStorage (never gated behind identity). (Implementation note: use a free markdown-input-rule editor framework — TipTap on ProseMirror — do NOT hand-roll contenteditable. The writing surface keeps the warm, calm, typographic page feel; the bare-text sidebar + char count + dateline chrome follow the SSENSE house system — monochrome ink/paper/grey, square corners, no shadows, no dividers.)
