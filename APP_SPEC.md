# Scratchpad
Purpose: for computer workers who need to jot a quick note or draft right now — a completely blank, instantly-typeable markdown page that formats itself as you type, keeps every note in a quiet left sidebar, and copies cleanly into Gmail/Docs/Notion — with nothing to set up.
Problem: software engineers, PMs, analysts, and marketers reach for a throwaway place to paste a snippet, draft a Slack message, or outline a thought several times a day. The usual fallbacks — a new untitled doc, a Notion page, a sticky-note app — make you pick a location, name a file, or stare at a toolbar before you can type. They want the cursor to already be blinking.
Beats alternative: vs Notion/Google Docs/Apple Notes, this app drops you ON a blinking cursor with ZERO setup (no new-doc dialog, no naming a file, no login) and live-formats markdown shortcuts (Dropbox-Paper style "## "→heading) as you type, so the very first keystroke is real writing — and a quiet SSENSE sidebar keeps every throwaway note you make, most-recent first, one click apart. That instant blank-canvas-with-live-markdown plus a frictionless note list is the reason this app exists.
Core flows:
1. New note → type immediately (instant-typing delight, every note). A quiet left sidebar has a "+ NEW NOTE" action at the top; clicking it creates a fresh blank note, makes it active, and focuses the caret so the first keystroke is real writing (zero clicks to type). First-ever load with no saved notes lands directly in a fresh blank focused note (original cold-open delight). When the user starts typing on a NEW blank note, today's date is auto-inserted ONCE as a quiet dateline ABOVE the first typed line (format "Saturday, June 20" — weekday, full month, day, no year/ordinal; captured at first keystroke, does not live-update, editable/deletable content, new notes only).
2. Live markdown input-rules as you type (the differentiator). Markdown shortcuts transform the current line/selection in place, consuming the syntax: "# " / "## " / "### " → H1/H2/H3; "- " → bullet; "1. " → numbered list; "> " → blockquote; "`code`" → inline code; "**bold**" → bold; "*italic*" → italic; "---" → horizontal divider. Pressing Enter after a heading returns the next line to a normal paragraph. A live character count sits in the lower-right of the writing surface, updating as you type.
3. Multi-note list + autosave (per-note localStorage). Each note is its own localStorage record (id, content, updatedAt); autosave is debounced and OFF the keystroke path. The sidebar lists every saved note most-recently-edited first, each row titled from the note's first non-empty line (fallback "Untitled"). Click a row to open/switch (the current note is saved first); the ACTIVE row is marked by inversion (ink fill). Delete a note via a quiet hover affordance + confirm. The last-open note is remembered across reloads. A whisper-quiet "Saved" affordance is the only save status shown.
4. Get your note out (single copy icon). Once the active note is non-empty, a single thin 1px line copy icon (in --ink) surfaces; clicking it copies the note as RICH TEXT (a ClipboardItem with text/html + a text/plain fallback) so paste into Gmail/Docs/Notion keeps formatting. There is NO markdown-text copy option. A brief monochrome "copied" confirmation shows in place (or an explicit "Copy blocked" failure), without stealing editor focus or shifting layout.
Success checks:
1. New-note focus, zero clicks: on first-ever load (no saved notes) OR immediately after clicking "+ NEW NOTE", WITHOUT clicking anywhere else, press the "h" key — "h" appears in the note and a text caret is visible. (Each new note is auto-focused.)
2. Heading rule: type exactly `## Roadmap` then no further typing — the line renders as a larger heading reading "Roadmap" with the leading "## " no longer visible anywhere on the page.
3. Bullet rule: on a new empty line type exactly `- buy milk` — the line renders as a bulleted list item showing a bullet marker followed by "buy milk", with "- " no longer visible as literal text.
4. Bold rule: type exactly `**ship it**` — the words "ship it" render in bold weight and the literal asterisks are no longer visible.
5. Divider rule: on a new empty line type exactly `---` — a horizontal rule (divider line) renders in place of the three dashes.
6. No layout jump: with the caret on a paragraph line, type `## ` to convert it to a heading — surrounding lines do not jump position (the line reformats in place; only that line changes size).
7. Comfortable measure: the writing column is centered with a constrained max width (roughly 65–75 characters per line, not full-bleed edge-to-edge).
8. Grotesque typeface: the editor writing surface and all chrome (sidebar, copy icon, saved indicator, char count, dateline) render in the SSENSE grotesque stack ("Helvetica Neue" → Archivo fallback), not a serif font.
9. SIDEBAR — new note + instant typing: clicking "+ NEW NOTE" adds a new row to the sidebar list, switches to it (it becomes the active/inverted row), and lands the focused caret in a blank surface so the next keystroke is writing — and the click does NOT get eaten by a prior surface's autofocus (the first keystroke after the click lands in the new note, not the old one).
10. SIDEBAR — title from first line + most-recent ordering: type `Sprint plan` as the first line of a note — its sidebar row title reads "Sprint plan"; a note whose first line is empty/blank shows "Untitled". Editing the first line updates the title. The most-recently-edited note appears at the top of the list.
11. SIDEBAR — active-by-inversion: the currently open note's row is marked by INVERSION (ink fill / paper text), with no color, no pill, no rounded corners; exactly one row is active at a time.
12. SIDEBAR — switch saves & restores (returning-user regression): create note A with `alpha note`, create note B with `bravo note`, click back to A's row — A opens showing "alpha note" (B's edits were saved first, A was not lost); reload the page — the last-open note re-opens and BOTH rows are still present in the list. (Covers multi-record persistence for a returning user, not just clean cold-open.)
13. SIDEBAR — delete with confirm: hovering a note row reveals a quiet delete affordance; activating it asks to confirm ("Delete this note?"); confirming removes that row from the list and from localStorage; the other notes remain.
14. COPY — single rich-text icon only: on a non-empty note a SINGLE thin copy icon (not two text actions) is visible; there is NO "Copy as Markdown" / markdown-text control or label anywhere in the UI or the built bundle (the old two-action affordance is fully removed, not just hidden).
15. COPY — writes HTML: type `**bold**` then click the copy icon — the clipboard contains a `text/html` flavor (verifiable via the async Clipboard API / pasting into a rich editor where "bold" renders bold), plus a `text/plain` fallback. No separate plain-markdown control exists.
16. COPY — perceptible confirmation surviving re-render: clicking the copy icon shows a monochrome "copied" confirmation that remains visible for at least ~1.5s (not a sub-150ms flash) and SURVIVES the autosave tick re-render; if the clipboard write fails/blocked it shows an explicit "Copy blocked …" message, never a silent success.
17. COPY — visibility + no focus-steal/jump: the copy icon is hidden on a blank note and appears once non-empty; clicking it does not move the caret out of the editor and does not shift the writing column or its lines.
18. CHAR COUNT — live + styled: with text in the note, a character count is visible in the LOWER-RIGHT of the writing surface and updates as you type (e.g. typing one more character increments it); it renders tiny, quiet, in --grey-600 with tabular-nums, monochrome (no fill/shadow).
19. DATELINE — auto on first keystroke, new notes only: click "+ NEW NOTE", then type `meeting notes` — a quiet --grey-600 dateline reading today's date in the exact form "Saturday, June 20" (weekday, full month, day; NO year, no ordinal suffix) appears as the line ABOVE "meeting notes", and typing continued on the line below; the dateline is editable/deletable real content. An untouched blank new note shows NO dateline until the first keystroke, and the dateline does not retroactively appear on previously-saved notes.
20. SNAPPY typing (off the keystroke path): typing does not serialize or persist the whole document on every keystroke — full-doc getJSON() / per-note save work happens only inside the debounced autosave timer; the per-keystroke path stays cheap (e.g. only the isEmpty boolean + character count update, char count derived cheaply), so typing has no perceptible lag.
21. MOBILE safe at 375px: at a 375px-wide viewport the sidebar collapses to a drawer (toggled by a quiet control) and the writing surface is full-width; when the drawer is open it does not permanently block the writing surface or controls, and when closed neither the drawer toggle, the copy icon, nor the char count overlays the caret or causes horizontal overflow.
Out of scope: no save/open dialogs or explicit note renaming (title auto-derives from the first line), no folders/tags/search across notes, no PDF/file export (clipboard rich-text copy IS supported), no markdown-text/plain-source copy option (rich-text only), no account or login, no server/Turso/cloud sync, no collaboration or sharing links, no toolbar or formatting buttons (formatting is keyboard markdown input-rules only). Notes are device-local in localStorage (never gated behind identity). (Implementation note: use a free markdown-input-rule editor framework — TipTap on ProseMirror — do NOT hand-roll contenteditable. The writing surface keeps the warm, calm, typographic Dropbox-Paper page feel; the sidebar + copy icon + char count + dateline chrome follow the SSENSE house system — monochrome ink/paper/grey, 1px hairlines, square corners, tiny uppercase tracked labels, no shadows.)
