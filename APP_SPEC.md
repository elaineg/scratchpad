# Scratchpad
Purpose: for computer workers who need to jot a quick note or draft right now — a completely blank, instantly-typeable markdown page that formats itself as you type, with nothing to set up.
Problem: software engineers, PMs, analysts, and marketers reach for a throwaway place to paste a snippet, draft a Slack message, or outline a thought several times a day. The usual fallbacks — a new untitled doc, a Notion page, a sticky-note app — make you pick a location, name a file, or stare at a toolbar before you can type. They want the cursor to already be blinking.
Beats alternative: vs Notion/Google Docs/Apple Notes, this app drops you ON a blinking cursor with ZERO chrome and ZERO setup (no new-doc dialog, no title, no sidebar, no login) and live-formats markdown shortcuts (Dropbox-Paper style "## "→heading) as you type, so the very first keystroke is real writing. That instant blank-canvas-with-live-markdown is the reason this app exists.
Core flows:
1. Cold load → type immediately. Page loads to a completely blank white writing surface with a faint placeholder hint and the editor already focused (blinking cursor, zero clicks). The first character typed appears in the document and dismisses the placeholder.
2. Live markdown input-rules as you type (the differentiator). Markdown shortcuts transform the current line/selection in place, consuming the syntax: "# " / "## " / "### " → H1/H2/H3; "- " → bullet; "1. " → numbered list; "> " → blockquote; "`code`" → inline code; "**bold**" → bold; "*italic*" → italic; "---" → horizontal divider. Pressing Enter after a heading returns the next line to a normal paragraph.
3. Debounced localStorage autosave round-trip. Edits are saved (debounced) to a single localStorage key; reloading the page restores the exact document. A whisper-quiet "Saved" affordance is the only status shown — no other chrome.
4. Get your note out (copy). Once the document is non-empty, a faint top-right corner control fades in (outside the writing column) with two labeled actions: "Copy as Rich Text" (writes a ClipboardItem with text/html + a text/plain markdown fallback, so paste into Slack/Gmail/Docs/Notion keeps formatting) and "Copy as Markdown" (copies the raw markdown source). The control is hidden on a blank document and hides again when emptied; clicking shows a ~1.8s in-place "Copied ✓" confirmation (or an explicit "Copy blocked" failure state) without stealing editor focus or shifting layout.
Success checks:
1. Cold-load focus, zero clicks: open the page in a fresh tab and, WITHOUT clicking anywhere, press the "h" key — the character "h" appears in the document and a text caret is visible. (Editor is auto-focused on load.)
2. Heading rule: type exactly `## Roadmap` then no further typing — the line renders as a larger heading reading "Roadmap" with the leading "## " no longer visible anywhere on the page.
3. Bullet rule: on a new empty line type exactly `- buy milk` — the line renders as a bulleted list item showing a bullet marker followed by "buy milk", with "- " no longer visible as literal text.
4. Bold rule: type exactly `**ship it**` — the words "ship it" render in bold weight and the literal asterisks are no longer visible.
5. Divider rule: on a new empty line type exactly `---` — a horizontal rule (divider line) renders in place of the three dashes.
6. Autosave round-trip: type the exact string `persist test 123`, wait 2 seconds, then reload the page (F5) — after reload the document still contains the text "persist test 123" (no clicks or re-typing needed).
7. No layout jump: with the caret on a paragraph line, type `## ` to convert it to a heading — surrounding lines do not jump position (the line reformats in place; only that line changes size).
8. No chrome on load: on cold load there is no visible toolbar, menu bar, header, sidebar, or buttons — only the blank centered writing surface (optionally a faint placeholder hint that disappears on first keystroke).
9. Comfortable measure: the writing column is centered with a constrained max width (roughly 65–75 characters per line, not full-bleed edge-to-edge).
10. Copy affordance visibility: on cold (blank) load NO copy control is visible (still passes check 8); after typing any non-empty text the two labeled actions "Copy as Rich Text" and "Copy as Markdown" become visible in the top-right corner; after selecting all and deleting back to empty, both actions are hidden again.
11. Copy as Markdown copies the source: type `## Title` then `- item`, click "Copy as Markdown", and paste into a plain-text field — the clipboard contains the raw markdown source (literal `## Title` and `- item`), not rendered HTML.
12. Copy as Rich Text writes HTML: type `**bold**` then click "Copy as Rich Text" — the clipboard contains a `text/html` flavor (verifiable via the async Clipboard API / pasting into a rich editor where "bold" renders bold), plus a `text/plain` fallback.
13. Perceptible copy confirmation: clicking either copy action replaces that action's label in place with a "Copied … ✓" confirmation that remains visible for at least ~1.5s (not a sub-150ms flash) and survives autosave re-render; if the clipboard write fails/blocked it shows an explicit "Copy blocked …" actionable message, never a silent success.
14. No focus-steal / no layout jump: with the caret in the editor, clicking a copy action does not move the text caret out of the editor and does not shift the position of the writing column or its lines.
15. Mobile safe at 375px: at a 375px-wide viewport the copy controls do not overlay the writing column or the caret and stay within the viewport (no horizontal overflow).
16. Grotesque typeface: the editor writing surface and chrome (copy buttons, saved indicator) render in the SSENSE grotesque stack ("Helvetica Neue" → Archivo fallback), not a serif font.
17. Snappy typing: typing does not serialize or persist the document on every keystroke — full-doc getJSON() work happens only inside the 800ms debounced autosave timer; the per-keystroke path only updates the cheap editor.isEmpty boolean, and only when its value actually changes.
Out of scope: no multi-note / "new note" management, no save/open/named notes, no sidebar or note list, no PDF/file export (clipboard copy IS supported via the copy affordance), no account or login, no server/Turso/cloud sync, no collaboration or sharing links, no toolbar or formatting buttons (formatting is keyboard markdown input-rules only). ONE scratchpad, single localStorage document. (Implementation note: use a free markdown-input-rule editor framework — TipTap on ProseMirror, or Lexical's markdown-shortcut plugin — do NOT hand-roll contenteditable. Reference aesthetic is DROPBOX PAPER (warm, calm, typographic, generous line-height, centered ~65–75ch measure); this OVERRIDES the SSENSE house chrome since there is essentially no chrome.)
