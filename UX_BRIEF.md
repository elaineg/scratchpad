# UX Brief — Scratchpad

## 1. Problem statement
A blank page that's already waiting for you — open it and just start typing, no doc to name, no toolbar to learn; it quietly formats and saves itself as you go, keeps every note in a calm sidebar, and copies cleanly into your email or doc.

## 2. Primary user action
**Type.** The cursor is already blinking, auto-focused, in the center column — on first load and on every "+ NEW NOTE". There is no button to press first to start writing, nothing to set up. The first keystroke is already real writing. A faint placeholder ("Start typing…") shows the intent and vanishes the instant a key is pressed.

## 3. Emotional tone
Warm, calm, focused — a quiet writing room, not an app. The WRITING SURFACE keeps the Dropbox Paper / iA-Writer feel (paper-white, not harsh #fff; soft near-black ink, not #000; generous airy spacing, unhurried). The CHROME around it — left sidebar, copy icon, char count, dateline — follows the SSENSE house system: monochrome ink/paper/grey, 1px hairlines, square corners, tiny uppercase tracked labels, no shadows. The two coexist: SSENSE is austere and recessive enough that the chrome stays understated and the surface stays the delight. Typography carries both.

## 4. Design decisions
- **Calm cold load, caret already blinking (make-or-break).** The writing surface is the hero: a centered writing column with a blinking auto-focused caret and a faint single-line placeholder. The ONLY chrome is the recessive SSENSE left sidebar (quiet hairline, tiny labels) — no toolbar, no formatting buttons, no menu bar, no logo, no header banner. First-ever load (no saved notes) lands directly in a fresh blank focused note. The value needs no instruction — you just type; the sidebar waits quietly for when you want a second note.
- **Reformat with zero jump (make-or-break).** Markdown input-rules are framework-native (TipTap/ProseMirror or Lexical markdown-shortcut). Typing `## ` consumes the syntax in place and the line reflows to a heading with NO page jump, NO caret jump, NO surrounding-line shift — only that one line changes size, caret stays put at the typing position. Keystrokes have zero perceptible lag; the caret is smooth and always visible.
- **Whisper-quiet autosave.** Debounced (~800ms) save to one localStorage key. The ONLY status affordance is a tiny low-contrast "Saved" word in a screen corner (bottom-right), light grey, small; it fades in briefly after a save settles and fades back to nothing. Default canvas shows nothing distracting. Reload restores the exact document.
- **Single copy icon, rich text only (REPLACES the old two text actions).** The prior "Copy as Rich Text" / "Copy as Markdown" labeled pair is REMOVED entirely — no markdown-text copy, no second control, no dead labels in the bundle. In its place a SINGLE thin 1px line copy icon (two-rectangle "copy" glyph, stroke `var(--ink)`, no fill, square corners, ~16px) sits in the TOP-RIGHT corner of the viewport, fixed, outside the writing column. It is HIDDEN on a blank note and FADES IN (~250ms opacity, no slide/no layout shift) once the note is non-empty; it fades out when emptied. Resting stroke is `--grey-600`, darkening to `--ink` on hover/focus. Clicking writes a `ClipboardItem` with `text/html` (rendered HTML — headings, bold, bullets, dividers) + a `text/plain` fallback, so paste into Gmail/Docs/Notion keeps formatting. Click does NOT move focus out of the editor (restore caret after the write) and does NOT shift the writing column.
- **Robust copy confirmation + explicit failure (honor transient-cue lesson).** On success the icon swaps IN PLACE to a thin `--ink` checkmark glyph with a tiny uppercase "COPIED" micro-label (`--fs-micro`, `--ls-micro`, `--ink`), monochrome — no color, no green — and HOLDS for ~1.8s (well above any sub-150ms flash, perceptible to a sighted user, survives the autosave re-render because the confirm state is keyed to the control, not a detached toast), then eases back to the resting icon. If the clipboard write throws/blocks, it instead shows "COPY BLOCKED — SELECT ALL & ⌘C" as a `--grey-800` micro-label for ~3s — explicit, actionable, never a silent pretend-success.
- **Quiet SSENSE left sidebar (understated, writing stays calm).** A fixed-width (~240px) left column, separated from the writing surface by a single 1px `--grey-200` hairline (vertical rule, no box, no shadow, square corners), `--paper` background. TOP: a "+ NEW NOTE" action as a tiny uppercase tracked label button (`--fs-label`, `--ls-label`, `--ink`, transparent bg) with a hairline-bottom separating it from the list — clicking it creates a fresh blank note, makes it the active row, and lands the focused caret in the surface so the next keystroke is writing (preserve zero-click typing; the click must NOT be eaten by the previous surface's autofocus — move focus to the new editor AFTER the new note mounts). UNDERNEATH: the saved-notes list, most-recently-edited first; each ROW shows a TITLE derived from the note's first non-empty line (fallback "UNTITLED"), `--fs-sm`, single-line truncated with ellipsis, generous vertical padding (`--sp-3`), separated by `--grey-200` hairlines (rows, never cards). The ACTIVE row is marked by INVERSION ONLY — `--ink` background, `--paper` text — no pill, no accent color, no rounded corners; exactly one row inverted at a time. Hover on an inactive row = `--grey-50` wash.
- **Delete affordance (quiet, confirm).** A note row reveals a thin 1px line "×"/trash glyph (stroke `--grey-600`, `--ink` on hover) on hover/focus only — hidden at rest so the list stays calm. Activating it asks to confirm inline ("DELETE THIS NOTE?" micro-label with a tiny CONFIRM / CANCEL pair, monochrome) before removing the note from the list and localStorage; other notes remain. On the active row the delete glyph reads in `--paper` against the inverted fill.
- **Live character count (lower-right of the surface).** A whisper-quiet count fixed in the LOWER-RIGHT corner of the writing surface, `--fs-micro` / `--grey-600` / `tabular-nums` (digits don't jitter), monochrome, no fill/shadow, optional tiny uppercase unit. It updates as you type (counts the active note) but is derived cheaply on the keystroke path — never via a full-doc serialize. It sits in the corner gutter, never overlapping the centered writing column.
- **Auto-dateline on new notes (calm, captured once).** When the user starts typing on a NEW blank note, today's date is auto-inserted ONCE as a line ABOVE the first typed line, formatted exactly `toLocaleDateString(undefined,{weekday:'long',month:'long',day:'numeric'})` → "Saturday, June 20" (no year, no ordinal). Style it quietly as a dateline: `--grey-600`, `--fs-sm`, regular weight, sitting above the body like a soft header — monochrome, no rule, no shouting. It is real editable/deletable content saved with the note; it does NOT live-update and is NEVER retroactively added to existing notes. The user keeps typing seamlessly on the line below — instant-typing delight intact.
- **Mobile (≤375px) safe.** The sidebar collapses to a DRAWER, off-canvas by default, with the writing surface full-width. A quiet 1px line hamburger toggle (stroke `--ink`) sits top-left; opening slides the drawer over the surface with a 1px `--grey-200` edge (no shadow); a tap on the surface / a quiet "×" closes it so it never permanently blocks the writing surface or controls. When the drawer is closed, neither the toggle, the copy icon (top-right), nor the char count (lower-right) overlays the caret or causes horizontal overflow; the dateline and body stay full-width within comfortable gutters.

## 5. Typography spec (the product)
- **Font surface:** humanist/readable stack — `ui-serif, "Iowan Old Style", "Charter", Georgia, "Times New Roman", serif` for body (Paper-warm). Acceptable alt: a humanist sans (`ui-sans-serif, "Inter", system-ui`). Builder picks one; commit to it.
- **Palette:** paper background `#FBFAF8` (warm near-white); ink `#2B2B29` (soft near-black, not #000); muted grey `#A6A39C` for placeholder + "Saved".
- **Body:** ~19px, line-height ~1.75, paragraph spacing ~1em between paragraphs.
- **Measure:** centered column, `max-width: 68ch` (within the 65–75ch target), comfortable top margin (~12vh) and side gutters; never full-bleed.
- **Heading scale:** H1 ~32px / lh 1.25; H2 ~26px; H3 ~21px; semibold, slightly tighter, with breathing room above.
- **Marks:** bullets = subtle round marker with hanging indent; numbered lists aligned; blockquote = soft left hairline rule (grey) + indent, ink slightly muted; inline `code` = light grey background, monospace, small radius; `---` divider = single faint hairline rule, generous space above/below; bold/italic real weight/slant.

## 6. 5-second check (above the fold, cold visitor)
- **Headline:** none rendered — the blank page IS the message.
- **Subtitle:** none.
- **Primary action:** blinking auto-focused caret in the centered column (zero clicks to type), with a quiet "+ NEW NOTE" label at the top of the recessive left sidebar.
- **Pre-filled example / hint:** faint grey "Start typing…" placeholder that disappears on first keystroke; on first keystroke a quiet grey dateline ("Saturday, June 20") appears above the line.
- **Chrome present but recessive:** thin sidebar hairline at left, copy icon (appears only once non-empty) top-right, live char count lower-right — all monochrome, none competing with the surface.
