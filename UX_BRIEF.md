# UX Brief — Scratchpad

## 1. Problem statement
A blank page that's already waiting for you — open it and just start typing, no doc to name, no toolbar to learn; it quietly formats and saves itself as you go, and keeps every note in a calm, chrome-free column of faint-grey titles.

## 2. Primary user action
**Type.** The cursor is already blinking, auto-focused, in the center column — on first load and on every "+ NEW NOTE". There is no button to press first to start writing, nothing to set up. The first keystroke is already real writing. A faint placeholder ("Start typing…") shows the intent and vanishes the instant a key is pressed.

## 3. Emotional tone
Warm, calm, focused — a quiet writing room, not an app. The WRITING SURFACE keeps the Dropbox Paper / iA-Writer feel (paper-white, not harsh #fff; soft near-black ink, not #000; generous airy spacing, unhurried). The CHROME around it — left sidebar, char count, dateline — follows the SSENSE house system: monochrome ink/paper/grey, square corners, no shadows. But the sidebar itself is now BARE TEXT, not a panel: no hairlines, no dividers, no fill — just faint-grey text floating on the same paper as the editor, so the chrome nearly disappears. Typography carries everything. The whole UI is set one weight notch thinner and ~20% smaller than before, for a lighter, calmer page.

## 4. Design decisions
- **Calm cold load, caret already blinking (make-or-break).** The writing surface is the hero: a centered writing column with a blinking auto-focused caret and a faint single-line placeholder. The ONLY chrome is the bare-text left sidebar (faint-grey lines, no panel) — no toolbar, no formatting buttons, no menu bar, no logo, no header banner. First-ever load (no saved notes) lands directly in a fresh blank focused note. The value needs no instruction — you just type; the sidebar waits quietly for when you want a second note.
- **Reformat with zero jump (make-or-break).** Markdown input-rules are framework-native (TipTap/ProseMirror or Lexical markdown-shortcut). Typing `## ` consumes the syntax in place and the line reflows to a heading with NO page jump, NO caret jump, NO surrounding-line shift — only that one line changes size, caret stays put at the typing position. Keystrokes have zero perceptible lag; the caret is smooth and always visible.
- **Whisper-quiet autosave.** Debounced (~800ms) save to one localStorage key. The ONLY status affordance is a tiny low-contrast "Saved" word in a screen corner (bottom-right), light grey, small; it fades in briefly after a save settles and fades back to nothing. Default canvas shows nothing distracting. Reload restores the exact document.
- **Bare-text left sidebar — faint-grey lines, NO panel chrome.** A fixed-width (~240px) left column with NO vertical hairline, NO border, NO box, NO background fill or shading — it is just a column of faint-grey TEXT LINES on the same `--paper` as the editor, styled like the old copy-affordance text (small, faint grey). It must NOT read as a panel. TOP: a `+ new note` action — LOWERCASE (no uppercase, no tracking), faint `--grey-600`, in the SAME typeface/size/weight as the list rows (a "+" prefix is fine; words stay lowercase). Clicking it creates a fresh blank note, makes it active, and lands the focused caret in the surface so the next keystroke is writing (preserve zero-click typing; the click must NOT be eaten by the previous surface's autofocus — move focus to the new editor AFTER the new note mounts). UNDERNEATH: the saved-notes list, most-recently-edited first. EVERY row uses the SAME typeface/size/weight/faint-grey as `+ new note` — one consistent text treatment for the whole sidebar (no hairlines or cards between rows). Each ROW's title derives from the first few phrases of the note's actual CONTENT that fit within ~20 chars, truncated cleanly, SKIPPING the auto-dateline line (the dateline is the literal first line, so derive the title from the first real typed text after it); fallback "Untitled" when there's no content. The ACTIVE row is marked by INK COLOR ONLY — active row renders in full `--ink`, every other row (and `+ new note`) stays faint grey — or at most a subtle weight bump. NO inversion, NO fill, NO shading, NO box, NO pill, NO underline-box; exactly one row is active at a time.
- **Delete — quiet "×", deletes immediately, NO confirm.** A note row reveals a quiet faint-grey "×" affordance (`--grey-600`, `--ink` on hover) on hover/focus only — hidden at rest, no button box, so the list stays calm. Activating it removes that note from the list and from localStorage IMMEDIATELY — NO "Delete this note?" dialog, NO confirm step; the other notes remain. Title auto-derives from content (editing the text renames; no explicit rename control).
- **Live character count (lower-right of the surface).** A whisper-quiet count fixed in the LOWER-RIGHT corner of the writing surface, `--fs-micro` / `--grey-600` / `tabular-nums` (digits don't jitter), monochrome, no fill/shadow, optional tiny uppercase unit. It updates as you type (counts the active note) but is derived cheaply on the keystroke path — never via a full-doc serialize. It sits in the corner gutter, never overlapping the centered writing column.
- **Auto-dateline on new notes (very faint, tight, captured once).** When the user starts typing on a NEW blank note, today's date is auto-inserted ONCE as a line ABOVE the first typed line, formatted exactly `toLocaleDateString(undefined,{weekday:'long',month:'long',day:'numeric'})` → "Saturday, June 20" (no year, no ordinal). Style it VERY FAINT and SMALL: `--grey-600`, font-size SAME as the body or SMALLER, NEVER a heading. NO blank line / NO vertical gap underneath it — the first typed line sits directly below the dateline, tight (no empty paragraph between them). It is real editable/deletable content saved with the note; it does NOT live-update and is NEVER retroactively added to existing notes. The user keeps typing seamlessly on the line below — instant-typing delight intact.
- **Mobile (≤375px) safe.** The bare-text sidebar collapses to a DRAWER, off-canvas by default, with the writing surface full-width. A quiet 1px line hamburger toggle (stroke `--ink`) sits top-left; opening slides the drawer (still bare faint-grey text, no panel chrome) over the surface; a tap on the surface / a quiet "×" closes it so it never permanently blocks the writing surface or controls. When the drawer is closed, neither the toggle nor the char count (lower-right) overlays the caret or causes horizontal overflow; the dateline and body stay full-width within comfortable gutters.

## 5. Typography spec (the product)
- **Font surface:** the SSENSE grotesque stack — `"Helvetica Neue", Helvetica, Arial, "Archivo", system-ui, sans-serif` for both surface and chrome (Archivo loaded via Google Fonts). Warm, calm page feel comes from palette + spacing, not a serif.
- **Global scale & weight (Elaine 2026-06-20):** the body/paragraph typeface is one weight notch THINNER (~400 → ~300/350, whatever the loaded Archivo/Helvetica weights support) for a lighter, calmer text color. EVERYTHING is ~20% SMALLER: reduce the root font-size ~16px → ~13px so all rem-based sizing (body, headings, sidebar, char count, dateline, spacing) scales down uniformly — proportions/rhythm stay intact, text stays legible. The sizes below are PRE-scale references; let the root reduction carry the 20%.
- **Palette:** paper background `#FBFAF8` (warm near-white); ink `#2B2B29` (soft near-black, not #000); muted grey `#A6A39C` for placeholder, "Saved", and the faint-grey sidebar text.
- **Body:** ~19px (pre-scale), line-height ~1.75, paragraph spacing ~1em between paragraphs, lighter weight per above.
- **Measure:** centered column, `max-width: 68ch` (within the 65–75ch target), comfortable top margin (~12vh) and side gutters; never full-bleed.
- **Heading scale (pre-scale):** H1 ~32px / lh 1.25; H2 ~26px; H3 ~21px; semibold, slightly tighter, with breathing room above.
- **Marks:** bullets = subtle round marker with hanging indent; numbered lists aligned; blockquote = soft left hairline rule (grey) + indent, ink slightly muted; inline `code` = light grey background, monospace, small radius; `---` divider = single faint hairline rule, generous space above/below; bold/italic real weight/slant.

## 6. 5-second check (above the fold, cold visitor)
- **Headline:** none rendered — the blank page IS the message.
- **Subtitle:** none.
- **Primary action:** blinking auto-focused caret in the centered column (zero clicks to type), with a quiet lowercase `+ new note` in faint grey at the top of the bare-text left sidebar.
- **Pre-filled example / hint:** faint grey "Start typing…" placeholder that disappears on first keystroke; on first keystroke a very faint, small grey dateline ("Saturday, June 20") appears tight above the first line.
- **Chrome present but near-invisible:** the left sidebar is bare faint-grey text (no hairline, no panel), live char count lower-right — monochrome, nothing competing with the surface. No copy control anywhere.
